# 贡献指南（CONTRIBUTING）

感谢你对本项目的关注和贡献！请按照以下规范参与协作。

## 如何贡献

- **Issue**：如发现 bug 或有新功能建议，请在 GitHub 上新建 issue，描述清楚问题和复现步骤。
- **Pull Request（PR）**：
  1. Fork 本仓库，并从 develop 或最新 feature 分支创建个人分支。
  2. 分支命名建议为 `feature-<你的名字>` 或 `fix-<你的名字>`。
  3. 按照项目代码风格进行开发，建议补充必要的测试。
  4. 确保所有代码通过测试和 lint 检查。
  5. 提交 PR 时请详细描述你的修改内容，并关联相关 issue（如有）。

## 代码规范

- 代码和注释请尽量清晰、易读。
- 遵循对应语言的主流风格（如 Python 遵循 PEP8，JS 遵循 StandardJS）。
- 保持函数和文件的单一职责，结构清晰。

## 提交信息规范

- 提交信息简明扼要，建议格式如：`fix: 修复数据解析 bug` 或 `feat: 新增用户登录接口`。

## 评审流程

- 所有 PR 都需由维护者审核。
- 请及时响应评审意见并修改代码。

## 其他说明

- 如需进行重大更改，请先通过 issue 讨论方案。
- 保持沟通礼貌、尊重他人。

感谢你的贡献，让项目更好！
